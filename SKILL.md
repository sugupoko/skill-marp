# Marp Skill

Marpを使ってMarkdownからスライドを作成するスキル。

## Quick Reference

| Task | Command |
|------|---------|
| HTMLプレビュー生成 | `marp slide.md -o slide.html` |
| PDF出力 | `marp slide.md -o slide.pdf` |
| PPTX出力 | `marp slide.md -o slide.pptx` |
| 画像出力 (各スライド) | `marp slide.md --images png` |
| テーマ付き出力 | `marp slide.md --theme theme-simple.css -o slide.html` |

---

## Marp Markdownの基本構文

```markdown
---
marp: true
html: true
paginate: true
---

<!-- _class: title -->

# プレゼンテーションタイトル

## サブタイトル
発表者名

---

# スライドタイトル

**リード文: このスライドで伝えたいことを1-2行で**

- 箇条書き1
- 箇条書き2
- 箇条書き3

---

<!-- _class: section -->

# セクション区切り

---

# 2カラムレイアウト

**リード文をここに書く**

<div class="columns">
<div>

### 左カラム
- ポイント1
- ポイント2

</div>
<div>

### 右カラム
- ポイント3
- ポイント4

</div>
</div>
```

**重要**: `html: true` をfront matterに必ず含めること。HTMLタグ（`<div>`等）が有効になる。

### スライド区切り

`---` (水平線) でスライドを区切る。

### ディレクティブ

| ディレクティブ | 説明 | 例 |
|---------------|------|-----|
| `marp: true` | Marp有効化 (必須) | front matterに記載 |
| `theme` | テーマ指定 | `theme: simple-jp` |
| `paginate` | ページ番号表示 | `paginate: true` |
| `_class` | スライド個別クラス | `<!-- _class: title -->` |
| `_backgroundColor` | 背景色 | `<!-- _backgroundColor: #1a1a2e -->` |
| `_color` | 文字色 | `<!-- _color: white -->` |
| `header` | ヘッダー | `header: 'プロジェクト名'` |
| `footer` | フッター | `footer: '© 2026'` |

### 画像の配置

```markdown
# 通常の画像
![](image.png)

# サイズ指定
![w:500](image.png)
![h:300](image.png)
![w:400 h:300](image.png)

# 背景画像
![bg](background.png)
![bg left:40%](image.png)   <!-- 左40%に画像、右60%にテキスト -->
![bg right:50%](image.png)  <!-- 右50%に画像、左50%にテキスト -->
![bg contain](image.png)    <!-- 背景にフィット -->
![bg fit](diagram.png)      <!-- アスペクト比維持でフィット -->
```

## テーマ

### 同梱テーマ: `simple-jp`

シンプルな日本語スライド用テーマ。MSPゴシックをデフォルトフォントとして使用。

**テーマファイル**: `theme-simple.css` (このスキルディレクトリに同梱)

**利用可能なスライドクラス:**

| クラス | 用途 | 背景 |
|--------|------|------|
| `title` | タイトルスライド | ダークネイビー (#1a1a2e) |
| `section` | セクション区切り | ダークグレー (#2d3436) |
| `summary` | まとめスライド | ダークネイビー (#1a1a2e) |
| (なし) | 通常コンテンツ | 白 (#FFFFFF) |

**使い方:**

```bash
marp slide.md --theme theme-simple.css -o slide.html
```

### カスタムテーマの作成

Marpテーマは `/* @theme テーマ名 */` で始まるCSSファイル。 `@import 'default'` で既定スタイルを継承可能。

## 作成ワークフロー

### 基本原則: タイトル + リード文 + コンテンツ

**全スライドの基本形は「タイトル」「リード文」「コンテンツ」の3段構成。**
これはPPTの定番構成であり、まずこの形で全スライドを組み立てる。
デザインパターン（比較、フロー等）は「コンテンツ部分の見せ方」として選択する。

```
┌──────────────────────────────────┐
│ h1: タイトル（トピックを端的に）     │  ← 固定位置のタイトルバー
├──────────────────────────────────┤
│ リード文（このスライドで言いたいこと  │  ← メッセージ=結論を1-2行で
│  を1-2行で。太字や強調で目立たせる） │
│                                  │
│  コンテンツ（箇条書き、表、図、     │  ← リード文を裏付ける根拠・詳細
│  2カラム、フロー図 など）           │
│                                  │
└──────────────────────────────────┘
```

**良い例:**
- タイトル: `検出精度が低い原因` → リード文: `一般人ラベラーの品質が低く、学習に悪影響を与えている` → コンテンツ: 専門家との比較データ
- タイトル: `最終ソリューション` → リード文: `疑似ラベル活用とYOLOパラメータ調整が差異化ポイント` → コンテンツ: 学習フロー図

**悪い例:**
- タイトルだけでリード文なし → 何が言いたいスライドか一目で分からない
- リード文が長すぎる(3行以上) → コンテンツと区別がつかない

**リード文が2行になる場合の注意:**
- リード文は中央揃え固定表示のため、折り返し位置が目立つ
- QAで画像を確認し、不自然な位置で改行されていたら `<br>` で明示的に改行位置を制御する
- 例: `**14の病理サンプル × 2種類のアノテータ + ラベル無しで構成。<br>データの扱いがコンペ最大のポイント。**`

**リード文が不要な場合:**
- まとめスライドなど、リード文なしでコンテンツだけ書きたい場合がある
- テーマCSSはh1直後の最初のpタグをリード文として扱うため、リード文なしで箇条書き等を直接書くとレイアウトが崩れる
- `&nbsp;` を空のリード文として挿入して回避する:

```markdown
# まとめ

&nbsp;

- ポイント1
- ポイント2
```

### 1. 要件整理

ユーザーの要望から以下を確認:
- スライドの目的・対象者
- 含めるべき内容
- 参考にするファイルがあるか
- 出力形式 (HTML / PDF / PPTX)

### 2. スライド構成を設計

各スライドについて以下を決める:
1. **タイトル**: トピックを端的に (→ h1になる。固定位置のタイトルバーに表示)
2. **リード文**: そのスライドで言いたいことを1-2行で (→ 太字のテキストになる)
3. **コンテンツ**: リード文を裏付ける根拠・詳細 (→ 箇条書き / 表 / 図 など)
4. **パターン**: コンテンツの見せ方 (箇条書き / 2カラム / 表 / フロー / 数値強調 など)

### 3. Markdownファイル作成

```markdown
---
marp: true
theme: simple-jp
paginate: true
---

<!-- _class: title -->

# プレゼンタイトル

## サブタイトル
発表者名

---

# タイトル（トピックを端的に）

**リード文: このスライドで伝えたい結論・主張を1-2行で書く**

- コンテンツ: リード文の根拠や詳細
- データや事実を並べる
- 必要に応じて表や図を入れる

---

# 次のスライドのタイトル

**次のリード文...**

次のコンテンツ...
```

### 4. 出力・確認

```bash
# HTMLで確認
marp slide.md --theme theme-simple.css -o slide.html

# 画像で確認 (QA用)
marp slide.md --theme theme-simple.css --images png
```

## レイアウトパターン

### タイトルスライド

```markdown
<!-- _class: title -->

# メインタイトル

## サブタイトル
発表者: 名前
```

### テキスト + 画像 (左右分割)

```markdown
# スライドタイトル

![bg right:45%](image.png)

- ポイント1
- ポイント2
- ポイント3
```

### 2カラムテキスト

```markdown
# スライドタイトル

**リード文をここに**

<div class="columns">
<div>

### カラム1タイトル
- 項目A
- 項目B

</div>
<div>

### カラム2タイトル
- 項目C
- 項目D

</div>
</div>
```

テーマCSSに定義済みのクラス:
- `columns` — 2カラム均等分割
- `columns3` — 3カラム均等分割

### 大きな数値の強調

```markdown
# 成果

<div class="center-stats">
<div>
<span class="big-number">10位</span><br>/ 1064チーム
</div>
<div>
<span class="big-number-gold">Gold</span><br>メダル獲得
</div>
</div>
```

テーマCSSに定義済みのクラス:
- `big-number` — 青い大文字 (3em)
- `big-number-gold` — ゴールドの大文字 (3em)
- `center-stats` — 数値を横並びで中央配置

### セクション区切り

```markdown
<!-- _class: section -->

# セクション名
```

### まとめスライド

```markdown
<!-- _class: summary -->

# まとめ

&nbsp;

- ポイント1
- ポイント2
- ポイント3
```

リード文不要の場合は `&nbsp;` で空のリード文を入れる。これがないと最初の箇条書きがリード文スタイルになってしまう。

## デザインパターン選定ガイド

スライドを作る前に、伝えたい情報の「関係性」からパターンを選ぶ。
参考: [パワーポイントのデザインパターン大全](https://speakerdeck.com/coneinc/pawapointonodezainpatanda-quan-zi-liao-zuo-cheng-shi-nishi-eru39noaidea)

### 要素間に関係性がある場合

| パターン | 使いどころ | Marp実装 |
|----------|-----------|----------|
| **並列** (横/縦) | 同列の項目を並べる | `display: flex` で横並び、箇条書きで縦並び |
| **比較** | 2案の対比、Before/After | 2カラム + テーブル |
| **フロー** | 手順・プロセス説明 | 番号付きリスト or `→` 記号で横フロー |
| **サイクル** | 繰り返しプロセス | テキスト + 矢印記号で表現 |
| **ピラミッド/ファネル** | 階層構造、絞り込み | インデント + 文字サイズ変更 |
| **マトリクス** | 2軸での分類 | 2x2テーブル |
| **ベン図/重なり** | 共通点・差異 | テキスト説明 or 画像 |
| **ツリー** | 組織図、分岐 | インデント付きリスト |
| **数式/加算** | A + B = C の関係 | 大文字 + 記号で表現 |

### 要素間に関係性がない場合

| パターン | 使いどころ | Marp実装 |
|----------|-----------|----------|
| **グラフ** | データの可視化 | 画像として挿入 |
| **スクリーンショット** | UI・画面の説明 | `![bg right:50%](screenshot.png)` |
| **テキストのみ** | 補足説明 | 箇条書き or blockquote |
| **Q&A** | FAQ形式 | `**Q:**` / `**A:**` の太字表記 |
| **数値強調** | KPI・成果 | `font-size: 3em` の大文字 |
| **引用/証言** | 権威付け | blockquote スタイル |

### ページ種別ごとの定番パターン

| ページ種別 | パターン |
|-----------|---------|
| **表紙** | `_class: title` — タイトル + サブタイトル + 発表者 |
| **目次** | 番号付きリスト or テーブル |
| **セクション区切り** | `_class: section` — セクション名のみ |
| **会社/プロジェクト概要** | 左テキスト + 右画像 (`bg right`) |
| **沿革/タイムライン** | 番号付きリスト or テーブル (年/出来事) |
| **メンバー紹介** | 2-3カラムの flex レイアウト |
| **実績** | 数値強調 + ロゴ画像 |
| **まとめ** | `_class: summary` — 箇条書き3-5点 |

### パターン選定の手順

1. **情報の関係性を見極める**: 並列？比較？フロー？関係なし？
2. **パターンを選ぶ**: 上の表から最適なものを選択
3. **Marp実装を確認**: 対応するレイアウトパターン（後述）を使う
4. **1スライド1パターン**: 複数パターンを1枚に詰め込まない

## デザインガイドライン

### フォント

- **デフォルト**: MSPゴシック (`theme-simple.css` に設定済み)
- コードブロック: Consolas / MS Gothic

### 文字サイズの目安

| 要素 | サイズ |
|------|--------|
| スライドタイトル | 1.6em (h1) |
| サブヘッダー | 1.3em (h2) |
| 本文 | 0.85em |
| キャプション | 0.7em |

### レイアウトの原則

- **シンプルに**: 1スライド1メッセージ
- **左揃え**: 本文は左揃え、タイトルのみ中央揃え可
- **余白を確保**: 詰め込みすぎない
- **一貫性**: 同じ種類のスライドは同じレイアウト
- **画像を活用**: テキストだけのスライドは避ける (可能な場合)

### 避けるべきこと

- 1スライドに情報を詰め込みすぎる
- フォントサイズが小さすぎる (0.7em未満)
- 背景色とテキスト色のコントラスト不足
- レイアウトの不統一

## QA (必須)

**最初のレンダリングはほぼ必ず問題がある。QAはバグ探しとして取り組むこと。**

### コンテンツ確認

Markdownファイルを直接読んで、内容の抜け・誤字・順序を確認。

### ビジュアル確認 (PDF → JPG → サブエージェントで目視)

```bash
# 0. QA用フォルダ作成
mkdir -p qa_img

# 1. PDF出力
marp slide.md --theme theme-simple.css -o slide.pdf --allow-local-files --no-stdin

# 2. PDF → JPG変換 (qa_img/slide-01.jpg, slide-02.jpg, ... になる)
pdftoppm -jpeg -r 150 slide.pdf qa_img/slide

# 3. 特定スライドだけ再変換する場合
pdftoppm -jpeg -r 150 -f N -l N slide.pdf qa_img/slide-fixed
```

**⚠️ サブエージェントを使う** — 自分で作ったスライドは期待通りに見えてしまう。サブエージェントに新鮮な目で確認させる。

サブエージェントへのプロンプト例:
```
これらのスライド画像を確認してください。問題がある前提で探してください。

確認ポイント:
- テキストがはみ出し・切れていないか
- タイトルバーが正しく上部固定されているか
- リード文がタイトルバーに重なっていないか
- リード文が2行に折り返している場合、改行位置は自然か（単語の途中で切れていないか、意味の区切りで改行されているか）
- 2カラムレイアウトが崩れていないか
- テーブルが見切れていないか
- フォントサイズが小さすぎないか
- 色のコントラストは十分か
- ページ番号が表示されているか
- 余白が十分か (端に詰まりすぎていないか)

各スライドについて問題をリストアップしてください:
1. /path/to/slide-01.jpg (タイトルスライド)
2. /path/to/slide-02.jpg (結果)
...
```

### 確認ループ

1. Markdown作成 → PDF出力 → JPG変換 → サブエージェントで目視確認
2. 問題をリストアップ (0件なら、もっと厳しく見直す)
3. 修正
4. 修正したスライドだけ再変換して再確認
5. 問題がなくなるまで繰り返す

**最低1回のfix-and-verifyサイクルを完了するまで完成とみなさない。**

## 出力形式

```bash
# HTML (ブラウザで閲覧)
marp slide.md --theme theme.css -o slide.html

# PDF
marp slide.md --theme theme.css -o slide.pdf --allow-local-files

# PPTX (PowerPoint)
marp slide.md --theme theme.css -o slide.pptx --allow-local-files

# PNG画像 (各スライド)
marp slide.md --theme theme.css --images png --allow-local-files

# JPEG画像
marp slide.md --theme theme.css --images jpeg --allow-local-files
```

`--allow-local-files` はローカル画像を含む場合に必要。
